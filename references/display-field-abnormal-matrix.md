# 前端外显字段 — 类型异常与展示形态矩阵（强制）

当接口响应节点中有字段会**正常展示在页面上**（前端外显），写用例时**必须**按类型做异常校验，并同时覆盖 **API 契约/造数** 与 **前端展示**（可拆成接口用例 + 前端功能用例，或联调用例）。

**原则：**

1. 先建 **字段映射表**：`接口路径.字段` → 类型 → 页面位置 → 展示形态（单行/多行/可滚动/弹层）→ 空值策略（隐藏模块 / 隐藏区块 / 兜底文案 / 待确认）。
2. 文档未写空值、少字段、多字段行为 → 预期写「记录实际 + 产品确认」，**禁止脑补业务规则**；但**用例必须存在**。
3. 异常校验**包含但不限于**下表；按模块实际字段裁剪，不可整类跳过「外显字段」。

---

## 1. 按类型的默认异常集

### 1.1 数值 `int` / `number`（外显）

| 场景 | 建议预期方向（有 PRD 则跟 PRD） |
|------|--------------------------------|
| 正数（正常） | 按设计展示，带单位/格式 |
| `0` | 展示 `0`（或约定单位）；**不等于**「无数据」；是否隐藏单项看产品 |
| 负数 | 不崩溃；展示负号 / 兜底 / 当 0 — 待确认；接口侧可另测是否应 4xx |
| 超大数 | 不溢出为 NaN/科学计数导致错乱；可省略号或完整展示 |
| 类型错乱（字符串数字 `"12"`、小数） | 前端可解析则展示；否则不白屏 — 记录实际 |

**最少用例：** 负数、0、正数 各至少 1 条（外显字段）。

### 1.2 字符串 `string`（外显）

| 场景 | 建议预期方向 |
|------|----------------|
| 正常文案 | 与接口一致展示 |
| 空字符串 `""` | 按区块策略：不展示该行/该文案 / 整模块隐藏 / 前端固定兜底 — **必须单独立项** |
| 仅空白 `"   "` | 是否 trim — 待确认；需有用例 |
| 超长 | 见「展示形态」：单行 `...` / 多行截断+展开 / 容器滚动 |
| 特殊字符 `!@#￥%…`、emoji、中英混排 | 正常展示、不乱码、不因解析当 HTML 执行 |
| HTML/脚本片段 `<script>`、`<>&` | **转义文本展示**，不执行脚本（安全展示） |
| 极短 1 字 | 布局不错位 |

**最少用例：** 空、超长、特殊字符 各至少 1 条。

### 1.3 数组 `[]`（外显列表/标签/多卡片）

| 场景 | 建议预期方向 |
|------|----------------|
| 空数组 `[]` | 整列表隐藏 / 空态 / 模块不展示 — 按 PRD |
| 1 个元素 | 展示 1 项；布局不按「满屏多列」错乱 |
| 少量（如 2～3，或「比设计稿少」） | 有几个展示几个；对齐规则（居左/居中）按设计 |
| 多个（正常满量） | 全部展示或分页/滚动 |
| 较多（如 10 个，或设计上限） | 不破版；可滚动 / 折行 / 「更多」 |
| 超过上限（若有 max） | 截断或滚动 — 待确认 |
| 元素缺字段 / 元素为 null | 单项跳过或占位 — 不白屏 |

**最少用例：** 空、1、少、多（含约 10 个）、正常多 中按场景覆盖；列表外显时 **空 + 1 + 多** 为 P0/P1。

### 1.4 对象 / 字典 `{}`（外显结构体，如状态区）

| 场景 | 建议预期方向 |
|------|----------------|
| 空对象 `{}` | 常等同「核心数据缺失」→ **整模块不展示**（若该对象是核心）— 与 PRD 对齐 |
| 仅含约定 key（正常） | 全量映射展示 |
| **少 key**（约定 N 个只回 M 个，M&lt;N） | **不报错**；有几个展示几个；布局自适应（如一行居左、不留永久大洞）— 以产品/实现为准，必须有用例 |
| **多 key**（约定外多余字段） | **多余 key 不展示**；不因未知字段崩溃；已知 key 仍正确 |
| key 在但 value 为 null | 按该 value 类型空值策略 |
| 嵌套层级错误（该是 object 给了 array） | 不白屏；模块隐藏或降级 — 记录实际 |

**最少用例：** 空 `{}`、少 key、多 key（未知字段）、正常满 key。

## 2. 展示形态（与类型正交，外显必问）

写用例前对每个外显字段标注 **展示形态**，并生成对应异常：

| 形态 | 典型控件 | 超长/过多时预期 | 空值时预期 |
|------|----------|-----------------|------------|
| **单行** | 时间、标签旁数字 | 超出一行加 `...` 省略；或整体缩小 — 按设计 | 整行不展示或占位 |
| **多行固定 N 行** | 摘要、说明 | 超出截断 + 展开/收起 | 同空策略 |
| **可滚动区域** | 弹层正文、长列表 | **固定高度**，内容超出可上下滑动，不撑破弹层 | 空内容 / 兜底文案 / 不打开 — 待确认 |
| **多列指标（object 多 key）** | 四项统计 | 少项时剩余项居左/均分 — 待确认；多项不换行挤爆则折行 | 全无则模块隐藏 |
| **弹层/半屏** | 数据来源说明 | 标题可前端写死；正文跟接口；超长滚动 | 正文空：兜底 or 空白 — **产品确认** |

## 3. 接口字段 → 前端外显 强制映射步骤

每个接口成功响应写完正向用例外，执行：

```text
FOR each data 节点字段:
  IF 该字段（或子字段）在 UI 有展示:
    标记 type = int|string|array|object|...
    标记 surface = 页面位置 + 展示形态
    生成「类型异常集」∪「展示形态集」用例
    标签建议: 前端;外显;异常;{字段名};{类型}
  ELSE:
    仅 API 契约/安全即可，不必做 UI 截断类用例
```

交付说明中建议附一张简表：

| 字段 | 类型 | 页面位置 | 展示形态 | 空值策略 | 已覆盖异常 |
|------|------|----------|----------|----------|------------|

## 4. 示例模式（共创看板类，可复用到类似模块）

> 下列为**模式示例**（数值/文案可替换）；具体预期以 PRD/产品确认+实现为准。

### 4.1 `coCreationStatus`（object，外显四项指标）

约定 key 示例：`userParticipationCount`、`adoptedCount`、`reviewingCount`、`updatedReportCount`。

| 用例意图 | 预期方向（示例） |
|----------|------------------|
| 返回**多余**未知 key/value | 仅展示约定 4 项；多余不展示；不报错 |
| 约定 key **少返回**几个 | 不报错；有几个展示几个；**同一行居左**（或设计约定对齐） |
| `coCreationStatus` 为 `{}` 或缺失 | **该模块不展示**（核心状态为空） |
| 四项均为合法 int（含 0/正） | 映射正确；0 展示为 0 项 |
| 某 count 为负数 | 不崩溃；展示策略待确认 |

### 4.2 `latestUpdate.latestUpdateTime`（string，底部单行）

| 用例意图 | 预期方向（示例） |
|----------|------------------|
| 空字符串 / null / 无 latestUpdate | **不展示**「最近更新 + 时间」整段（模块其余可在） |
| 特殊字符 `!@#gj..` | 能正常展示（或过滤后展示），不白屏 |
| 文案特别长 | **占一行**，超出 `...` 省略 |
| 正常时间格式 | 按设计格式化展示 |

### 4.3 `sourceText`（string，弹层正文；标题可前端写死）

| 用例意图 | 预期方向（示例） |
|----------|------------------|
| 超长 | 弹窗**固定高度**，正文**可上下滑动**查看 |
| 空字符串 | 标题仍为前端固定文案；正文是**前端兜底默认文案**还是**空白** → **产品确认**（用例必须写上两种可能+待确认） |
| 特殊字符 `!@#gj..` | 正常展示、转义安全 |
| 含 HTML | 当文本展示，不执行 |

## 5. 优先级建议

| 优先级 | 覆盖 |
|--------|------|
| P0 | 空对象/空数组导致模块或核心区隐藏；空字符串隐藏单行区块；正常映射 |
| P1 | 少 key/少元素布局；多 key 忽略；0 与正数；特殊字符；单行省略；弹层滚动 |
| P2 | 负数、超大数、约 10 条列表、空白 trim、类型弱校验 |
| P3 | 极端长度性能、冷门组合 |

---

## 6. 与分层交付的关系

- **接口用例**：负责 mock/返回上述结构时 **HTTP 不 500、字段可解析**；可选校验网关对非法类型的 returncode。
- **前端功能用例**：负责 **显隐、布局、省略、滚动、忽略未知字段、安全展示**。
- 用户只要「接口用例」时：仍应在备注/附录列出「外显字段前端异常清单」，或单独一节「建议前端补测」。
- 用户只要「前端用例」时：前置条件写清如何 mock 各异常响应。
