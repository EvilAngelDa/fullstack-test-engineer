# 响应场景分类：风控 ≠ 网络异常 ≠ 接口错误 ≠ 错误数据 ≠ 数据为空

写 **接口用例** 与 **前端显隐/恢复** 时，下列场景**必须拆开**，禁止合并成一条「异常就不展示」。

> 前端最终都可能「不展示模块」，但 **前置、响应壳、是否 toast、是否可恢复、是否影响兄弟模块** 不同 → **用例 ID / 场景必须分开**。

---

## 1. 场景对照表（泛化）

| 场景 | 典型特征（需按项目文档确认） | 测试要点 | 禁止与…混为一谈 |
|------|------------------------------|----------|-----------------|
| **风控 / 策略下线** | 业务成功码（如 `returncode=0`）+ **空业务体**（如 `data:{}`）；常按 **实体 ID 维度** 命中 | 成功壳 + 空 data；前端当「无数据」隐藏；**不**当传输失败；兄弟模块可仍有数据 | 网络失败、4xx/5xx、校验错误、串号、列表 `[]` |
| **网络异常** | 超时、断网、DNS、无响应、网关层失败 | 前端失败/重试/隐藏；恢复后可再拉有数据 | 风控成功空 data |
| **接口内部错误** | 如 `returncode=500` 或 HTTP 5xx；message 系统异常 | 与成功空 data 区分；不把 500 当成风控 | 风控 0+空 data |
| **参数/协议错误** | 如 `returncode=400`、缺参、非法参 | 校验矩阵；前端不展示业务模块 | 风控、空列表业务 |
| **错误数据** | 成功或失败壳下 **语义错误**：实体 ID 与请求不一致（串号）、类型错乱、字段矛盾 | 一致性/防串号；≠ 风控下线 | 风控、单纯空列表 |
| **数据为空（业务空）** | 成功码下 **结构化空**：字段在但列表 `[]`、或关键展示字段全空 | 前端无数据隐藏；与 `data:{}` **可并存为两种 mock** | 风控（可能同为成功空，但 **来源与造数不同**，用例前置要分开） |
| **实体不存在 / 无配置** | 文档定义的无资源、无配置（可能 0+空 或业务码） | 单独场景；文档未写则 待确认 | 默认等同风控 |

---

## 2. 空成功体的两种形态（都要测，都≠失败）

同一产品里「无展示内容」至少区分：

1. **极简空**：`data` 为 `{}`（常见于风控/策略直接短路）  
2. **结构化空**：`data` 有 key，但列表为空、字符串为空等（业务无内容）

规则：

- 前端：**有可展示数据才展示，否则不展示**（若产品如此约定）。  
- 用例：**风控命中**、**结构化空**、**双侧 `[]`** 分前置；不要删其中一个当重复。  
- 接口：两种 body 都 `returncode=0` 时，都要与 **400/500** 对照。

## 3. 用例拆分清单（生成时自检）

当需求出现「风控 / 下线 / 开关（服务端）」时，至少考虑：

| # | 场景 | API | FE |
|---|------|-----|----|
| A | 风控命中（成功+空 data） | ✓ | ✓ 不展示 |
| B | 未命中有数据 | ✓ 对照 | ✓ 展示 |
| C | 网络异常 | 可选造障 | ✓ 不展示/失败态；恢复 |
| D | 500 / 内部错误 | ✓ | ✓ 不展示；≠ 风控 toast 策略 |
| E | 400 / 参数错误 | ✓ | ✓ |
| F | 错误数据（串号、矛盾字段） | ✓ | ✓ 不串展示 |
| G | 业务空列表 / 结构化空 | ✓ | ✓ 不展示 |
| H | 实体不存在 | ✓ 若文档有 | ✓ |
| I | 跨模块：本模块风控空，兄弟模块有数据 | — | ✓ 兄弟仍展示 |
| J | 跨模块：本模块 500/网络失败，兄弟仍展示 | — | ✓ **与 I 分开** |

**禁止**：用一条「异常不展示」覆盖 A+C+D+E+F+G。

---

## 4. 记忆与评审用语（写入 memory 时）

优先使用下列 **泛化句**（勿写项目 case ID）：

- Risk control may return business success with empty data object; do not treat as network or 5xx failure  
- Network error, 4xx/5xx, wrong entity data, and empty business lists are separate scenarios even if UI all hides the module  
- Empty `data:{}` and structured empty (keys present, empty arrays) both need cases; neither equals transport failure  
- Sibling module isolation: risk empty-data vs server error must be separate FE cases  

---

## 5. 与平台用例维护

合并/删除平台用例时：

- 仅当 **场景完全等价** 才合并。  
- 风控成功空 data **不能** 替代：网络异常、500、400、串号、不存在、业务 `[]`。  
- 前端开关废弃 ≠ 删除「失败恢复」「跨模块失败隔离」类用例。
