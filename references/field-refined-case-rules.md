# 前端接口字段精细化用例强制规则

> 来源：用户专属约束 Prompt（沉淀为 skill 参考）。  
> **与旧规则关系：只增补、不删旧。** 与 `display-field-abnormal-matrix.md`、`api-fe-layer-split.md`、`response-scenario-taxonomy.md` 叠加使用。  
> **产品模块 PRD 显式约定优先于本文件默认兜底**（见文末冲突项，由用户裁定）。

---

## 1. 总规则

1. 针对**后端返回且前端渲染展示**的响应字段：**禁止**按数据类型笼统批量写一条用例。  
2. **必须逐字段拆解**，结合产品原型 / UI 设计稿真实展示逻辑，写**独立**校验用例。  
3. 每个字段严格匹配自身数据类型，覆盖：正常、边界、异常（按类型明细）。  
4. 凡绑定前端展示的字段用例，须落地 **「接口返回值 + 前端页面最终展示」** 双重预期（见 §2 分层写法）。  
5. **客户端展示用例** 与 **后端接口返回校验用例** **分开罗列**（两套清单/两个文件或明确分区）。  
6. **一条用例 = 一个字段 + 一个场景**，不合并多场景、不写「所有字符串统一校验超长」类概括用例。  
7. 必须标注：**字段全路径名** + **当前场景取值** + **前端 UI 依据**（`UI设计稿要求：…` 或 `产品需求约定：…`）。

---

## 2. 与「接口 / 前端分层」的兼容写法（不推翻旧分层）

| 用例归属 | 【接口返回预期】 | 【前端页面渲染预期】 |
|----------|------------------|----------------------|
| **接口用例（字段契约）** | 必填：码、类型、字段有无/取值集合 | 填 **「见前端字段用例 xxx」** 或不写样式细节；**禁止**写吸顶/横滑/字号等纯样式 |
| **前端用例（字段展示）** | 必填：本场景接口 mock/返回值（含字段路径） | 必填：页面最终展示/交互（截断、兜底、隐藏、滑动等） |

即：仍 **分开罗列** API 套件与 FE 套件；精细化要求的是 **FE 字段用例双段预期**，以及 **API 按字段拆场景**，而不是把样式写进 API 套件。

---

## 3. 字符串（前端外露）— 每字段至少 4 条独立用例

| 场景 | 接口侧 | 前端侧（须写明 UI 依据） |
|------|--------|--------------------------|
| 正常字符串 | 返回业务合规文本 | 完整原样展示；字体/换行/间距与 UI 稿一致 |
| 超长字符串 | 返回完整超长内容 | 按设计：单行打点 / 多行折叠 / 展开 / 弹窗全量 / 滚动等 |
| 特殊字符 | 含 `!@#$%^&*()_+-=`、中英符号、顿号、引号、emoji、中文标点等 | 无乱码、无符号丢失、无样式崩坏；原样展示（安全转义场景另遵安全规则） |
| 空值 | **分两条**：`""` 与 `null` | 按设计空白 / 横线 /「无」「暂无」等；不报错、不组件塌陷 |

---

## 4. int（前端外露）— 每字段至少 4 条

| 场景 | 要求 |
|------|------|
| =0 | 返回 0；默认前端展示数字 0（**若 PRD 写 0 不展示，以 PRD 为准，见冲突**） |
| 正常正数 | 原样展示；货币/数量匹配千分位、单位后缀（UI） |
| 合法负数 / 非法负数 | 业务允许则负号正常；禁止则兜底/报错 |
| 超大边界 | 不溢出为错误科学计数；按设计缩小字号/换行/横滑等 |

---

## 5. 数组 []（前端外露）— 至少 3 大场景 + 嵌套穿透

| 场景 | 要求 |
|------|------|
| 多元素（≥2） | 写明横向滑或竖向列表；**顺序与接口一致**；子项全渲染；滑动正常 |
| 仅 1 条 | 无多余空白占位、不多余滑动箭头；单条布局适配 |
| 空数组 `[]` | 默认兜底「暂无数据」类（**若 PRD 写整模块不展示，以 PRD 为准，见冲突**） |

数组内嵌套 string/int/object：对**每个子字段**再套对应类型规则。

---

## 6. 对象 {}（前端外露）

| 场景 | 要求 |
|------|------|
| 多 key（≥2） | 子 key 值均渲染；排版与设计一致 |
| 单 key | 单字段布局；无多余空白分区 |
| 空 `{}` | 兜底提示；不塌陷、不 JS 报错（核心数据空藏整模块时按 PRD，见冲突） |

内层每个 key 继续套 string/int/array 规则。

---

## 7. 额外兜底

1. **嵌套全穿透**：数组套对象、对象套数组，底层基础类型边界不跳过。  
2. **禁止批量概括**：必须 `字段全路径 + 场景值 + UI 依据`。  
3. **设计稿强绑定**：滚动/截断/排布/兜底文案每条备注 UI 或 PRD 依据。  
4. **布尔 / 浮点 / 时间**：true/false；正负 0 多位小数；正常/极端/空/特殊格式时间。  
5. **与场景分类叠加**：风控成功空、网络失败、4xx/5xx、错误数据仍须分开（`response-scenario-taxonomy.md`）。

## 9. 默认规则 vs 模块 PRD 冲突（须用户裁定）

下列默认值可能与具体模块 PRD 冲突；**模块 PRD 显式约定优先**，冲突时不得静默删本默认，应在用例备注「PRD 覆盖默认矩阵」：

| # | 本 Prompt 默认 | 常见模块 PRD 覆盖例 |
|---|----------------|---------------------|
| C1 | int=0 必须展示数字 0 | 统计类「0 或空不展示该字段」 |
| C2 | 空数组展示「暂无数据」兜底 | 「无数据不展示整模块」 |
| C3 | 空 `{}` 兜底提示 | 「核心空不展示整模块」 |
| C4 | 每条字段用例都含前端渲染段 | 纯接口套件仅契约（本 skill 用 §2 兼容表：API 用例前端段可引用 FE 用例） |

**请用户确认 C1–C4 全局默认：A=始终本 Prompt 默认优先；B=始终模块 PRD 优先（推荐）；C=其它。**  
在用户确认前，执行策略：**B — 模块 PRD 显式约定优先，默认矩阵用于 PRD 未写处。**
